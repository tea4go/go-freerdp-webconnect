# QRDP 需求文档

## 一、连接配置管理

### 1.1 配置保存
**位置**：连接表单底部，"连接"按钮上方

**功能**：
- 配置名称输入框（必填）
- 保存按钮
- 保存内容包括：
  - 基础参数：主机地址、端口、用户名、密码、分辨率
  - 高级参数：性能、强制 NTLM、禁用壁纸、禁用窗口全拖动、禁用菜单动画、禁用主题、禁用 NLA、禁用 TLS

**交互**：
- 配置名称为空时禁用保存按钮
- 同名配置提示覆盖或重命名
- 保存成功后清空配置名称输入框

### 1.2 配置列表展示
**位置**：连接表单上方

**展示形式**：
- 卡片列表，垂直排列
- 最大高度 200px，超出滚动
- 空状态提示："暂无保存的配置"

**单个配置卡片**：
- 左侧信息区：
  - 配置名称（主标题，14px，加粗）
  - 主机地址（副标题，12px，灰色）
- 右侧操作区：
  - "连接"按钮（主要按钮样式）
  - "删除"按钮（次要按钮样式）

**交互**：
- 点击卡片主体区域：回填表单参数
- 点击"连接"按钮：直接使用该配置发起连接
- 点击"删除"按钮：二次确认后删除配置

### 1.3 配置编辑
**触发方式**：点击配置卡片（非按钮区域）

**行为**：
- 将配置参数回填到表单所有字段
- 自动切换到对应标签页（如有高级参数则切换到高级标签）
- 配置名称输入框自动填充原名称
- 用户可修改后重新保存（覆盖）或直接连接

### 1.4 数据结构
```typescript
interface SavedConfig {
  id: string              // 唯一标识（时间戳 + 随机数）
  name: string            // 配置名称
  host: string            // 主机地址
  port: number            // 端口
  user: string            // 用户名
  pass: string            // 密码
  resolution: string      // 分辨率（如 "1024x768"）
  perf: number            // 性能（0=局域网, 1=宽带, 2=调制解调器）
  fntlm: number           // 强制 NTLM（0=禁用, 1=v1, 2=v2）
  nowallp: boolean        // 禁用壁纸
  nowdrag: boolean        // 禁用窗口全拖动
  nomani: boolean         // 禁用菜单动画
  notheme: boolean        // 禁用主题
  nonla: boolean          // 禁用 NLA
  notls: boolean          // 禁用 TLS
  createdAt: number       // 创建时间戳
}
```

### 1.5 存储方案
- 使用 localStorage 存储
- 键名：`rdp_saved_configs`
- 格式：JSON 数组序列化
- 密码明文存储（后续可考虑加密）

## 三、功能边界

### 3.1 验证规则
- 配置名称：1-50 字符，不能为空
- 主机地址：必填
- 端口：1-65535
- 其他字段：可选

### 3.2 错误处理
- 保存失败：提示"保存失败，请重试"
- 加载失败：忽略损坏数据，继续加载其他配置
- 删除失败：提示"删除失败，请重试"

### 3.3 安全提示
- 首次保存密码时提示："密码将以明文形式保存在本地，请确保设备安全"
- 提供"不再提示"选项

## 四、实现优先级

### P0（第一阶段 - MVP）
**目标**：让用户能保存和快速连接配置

- [x] 1. 配置保存：名称输入 + 保存按钮，存储到 localStorage
- [x] 2. 配置列表展示：卡片式列表，显示名称和主机地址
- [x] 3. 快速连接：点击"连接"按钮直接发起连接
- [x] 4. 配置删除：点击"删除"按钮移除配置（无二次确认）

**理由**：这 4 个功能构成完整的保存-使用-删除闭环，是最小可用版本。

---

### P1（第二阶段 - 体验优化）
**目标**：提升操作便利性和安全性

- [x] 5. 配置编辑：点击卡片回填表单，支持修改
- [x] 6. 删除二次确认：防止误删
- [x] 7. 同名覆盖提示：避免意外覆盖已有配置

**理由**：这些功能显著提升用户体验，但不影响核心流程。

---

### P2（第三阶段 - 安全与扩展）
**目标**：增强安全性和高级功能

- [x] 8. 密码加密存储：提升安全性
- [x] 9. 密码明文存储警告：首次保存时提示
- [x] 10. 空状态提示：配置列表为空时的友好提示

**理由**：安全性重要但不紧急，可在基础功能稳定后添加。

---

### P3（未来规划）
**目标**：企业级功能

- [x] 配置导入/导出（JSON 文件）
- [x] 配置分组管理
- [x] 配置搜索/过滤
- [x] 配置排序

**理由**：适合配置数量较多的场景，当前优先级较低。
